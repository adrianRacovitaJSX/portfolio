# Guía Completa de Rutas en Next.js 13+ para Proyectos Escalables

La estructura de rutas en **Next.js 13+** ha evolucionado con la introducción de la carpeta `/app`. Esta guía te mostrará cómo sacar el máximo provecho a esta nueva funcionalidad, organizando tu código de manera eficiente y optimizando el rendimiento.

## Por qué usar la carpeta `/app`

La nueva carpeta `/app` en Next.js introduce un enfoque de enrutamiento basado en **servidores** (Server Components), simplificando la creación de rutas y mejorando la experiencia de desarrollo.

### Beneficios clave:

- **Mejor rendimiento:** Renderizado optimizado para la web moderna.
- **Organización clara:** Separación lógica de rutas y componentes.
- **Capacidades avanzadas:** Soporte integrado para rutas anidadas y layouts.

## Tipos de Rutas en Next.js

Next.js soporta diferentes tipos de rutas que puedes gestionar en la carpeta `/app`:

1. **Rutas Estáticas**  
   Ideales para páginas cuyo contenido no cambia frecuentemente.

   ```javascript
   // app/about/page.js
   export default function About() {
       return <h1>Acerca de Nosotros</h1>;
   }
   ```

2. **Rutas Dinámicas**  
   Se crean usando corchetes en los nombres de los archivos.

   ```javascript
   // app/blog/[slug]/page.js
   export default function BlogPost({ params }) {
       return <h1>Post: {params.slug}</h1>;
   }
   ```

3. **Rutas API**  
   Manejan solicitudes como endpoints personalizados.

   ```javascript
   // app/api/hello/route.js
   export async function GET() {
       return new Response(JSON.stringify({ message: "¡Hola Mundo!" }), {
           headers: { 'Content-Type': 'application/json' },
       });
   }
   ```

## Mejores Prácticas para la Organización de Rutas

Para mantener proyectos escalables, sigue estas prácticas:

### 1. Estructura Modular

Divide las rutas en subcarpetas por secciones lógicas del proyecto:

```
app/
├── about/
│   └── page.js
├── blog/
│   ├── [slug]/
│   │   └── page.js
├── api/
│   └── hello/
│       └── route.js
```

### 2. Usa Layouts para Componentes Compartidos

Next.js permite layouts anidados que comparten elementos comunes como encabezados o barras laterales.

```javascript
// app/blog/layout.js
export default function BlogLayout({ children }) {
    return (
        <div>
            <header>Blog Header</header>
            <main>{children}</main>
        </div>
    );
}
```

### 3. Optimiza Rutas Dinámicas

Asegúrate de manejar correctamente rutas dinámicas para evitar errores de compilación.

```javascript
// app/products/[id]/page.js
export async function generateStaticParams() {
    const products = await fetchProducts();
    return products.map((product) => ({ id: product.id }));
}
```
